## 1. 引言

### 1.1 项目名称
**PDF AI 阅读器** (英文: PDF AI Reader)

### 1.2 产品目标
本软件旨在成为专业读者的终极研究助手，彻底改变"阅读-翻译-理解"的割裂流程。它以 **"AI 系统化内化全文 → 本地知识库存储 → 精确对照翻译 → 沉浸式裂缝交互问答"** 为闭环，帮助理学研究者以母语思维直接穿透英文论文的认知壁垒。核心不是翻译器，而是**以论文为单位的 AI 内化引擎**。

### 1.3 目标用户
- **主要用户**：数学、物理、计算机、人工智能、机器学习等领域的本科高年级学生、硕博研究生、科研人员、高校教师。
- **典型场景**：精读领域奠基性论文（如 "Attention Is All You Need"）、啃读数百页英文专业书籍、快速掌握某篇论文的证明脉络、撰写文献综述时对照提取关键论点。

### 1.4 核心价值
- **从"机翻"到"内化"的范式升级**：AI 不仅翻译，而是先批量深度解析全文，建立向量知识库。此后每一次提问、每一次翻译，都基于对全文的融会贯通，而非孤立句子。
- **"裂缝式"零干扰交互**：独创动态嵌入界面，在原文段落间直接"裂开"交互区，翻译、解释、问答就地呈现，彻底杜绝传统侧边栏或弹窗造成的视焦跳转与原文失联。
- **绝对隐私与离线能力**：默认内置轻量本地模型（Qwen3.5:4b），所有文档解析、知识库构建、基础翻译均在本地完成。云端模型仅作为增强选项，且每次调用均需用户显式授权。
- **专业语境铁壁守护**：数学公式绝对原样保留，定理、证明等结构获得特殊保护；强制术语表与翻译风格工程化，杜绝"不人话"直译，达到信达雅。

### 2.2 AI 全文理解与知识库构建

|------|------|---------|------|

### 2.4 裂缝式交互（核心交互）

裂缝是软件的**灵魂**——它不是弹窗，而是直接插入到 PDF 内容流中的临时动态交互容器。

#### 2.4.1 裂缝形态
- **展开态**：全宽视图，蓝紫渐变背景 + 圆角边框。包含标题栏、上下文指示器、输入区（提问模式）、QWebEngineView 结果区（Markdown + LaTeX 渲染）、追问建议按钮、操作栏（复制/重新生成/清除）。
- **折叠态**：完全隐藏——页面上下段无缝合并，段落左侧显示淡蓝色翻译指示条。
- **切换方式**：双击裂缝自身 / 点击 ∧ 按钮 / 按 Esc 键。

#### 2.4.2 触发机制
- 双击段落 → 首次翻译，再次折叠/展开切换
- 右键段落 → "翻译段落" / "在此处提问" / "解释此概念"

#### 2.4.3 交互行为
- 翻译模式：直接显示译文（不显示输入框）
- 提问模式：显示输入框 → 支持 Ctrl+Enter 发送 → 流式显示回答
- 追问建议：回答完成后自动生成 3 个可点击追问问题
- 多轮对话：保留最近 6 轮对话历史
- 底部拖拽手柄：可手动调整裂缝高度（拖拽后锁定自动高度）
- 自动撑高：内容超出当前高度时自动扩展（最大 600px）

## 3. 非功能需求

### 3.1 性能（实测）
- 百页文档任意滚动无卡顿（虚拟视口懒加载）
- 冷启动至主界面可见：≤2 秒
- 百页文档首屏解析至可交互：≤3 秒（阶段一启发式分块）
- ML 公式精扫：后台异步，不阻塞阅读
- 知识库构建：~1-3 分钟（百页论文，BGE-M3 逐条嵌入）
- 内存占用：应用 + 渲染 ≤800MB；含 4B 本地模型总计 ≤12GB

### 3.2 可用性
- 核心翻译和提问操作均不超过两次点击（右键 → 选择）
- 首次启动自动检测本地模型状态，引导安装
- 错误处理：网络/模型失败提供清晰提示

### 3.3 稳定性与可维护性
- 全局异常钩子：未捕获异常写入日志，弹简化错误对话框
- 日志系统：所有 AI 调用和耗时记录到 `logs/app.log`
- 模块化三层架构（ui/core/data），依赖注入

### 3.4 安全与隐私
- 本地内容绝不上传，除非用户显式配置云端 API Key
- API Key 存储在 `config.yaml`（可通过 .env 替代）
- 本地 Ollama 通信仅通过 `localhost:11434`

### 3.5 可移植性
- 当前目标平台：Windows 10/11（通过 conda 环境）
- macOS / Linux 计划后续支持

## 4. 数据需求
- **知识库**：ChromaDB 持久化至 `data/knowledge_bases/`，每个文档一个 Collection
- **术语表**：JSON 格式，结构 `{"domain": "...", "terms": [{"en":"...","zh":"...","force":true}]}`
- **配置**：`config.yaml`，支持 GUI 修改（简化版）和手工编辑
- **日志**：`logs/app.log`，含 AI 调用耗时和错误堆栈

## 5. 接口需求
- **LLM 抽象层**：`BaseLLMClient.generate(messages) → str` 和 `generate_stream(messages) → Generator[str]`
- **OllamaClient**：本地 REST API 客户端
- **LiteLLMClient**：云端统一客户端（支持 100+ 模型，内置 3 次自动重试）
- **MockLLMClient**：测试/降级用模拟客户端
- **HybridModelRouter**：任务路由决策
- **TranslationService**：公式保护 → 术语注入 → 风格控制 → LLM 调用 → 公式恢复
- **QAService**：知识库检索 → 上下文组装 → 多轮对话 → LLM 调用
