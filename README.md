# PDF AI Reader

> Attention is all you need.  
> 但读论文时，你还需要翻译、提问、公式、证据和一个不打断思路的阅读空间。

PDF AI Reader 是一个面向学术论文的智能 PDF 阅读器，旨在把论文从一份静态文件变成一个可以边读、边问、边理解的研究工作台。

你可以像普通 PDF 阅读器一样打开论文、滚动、缩放和跳转；也可以直接双击段落翻译，围绕全文提问，让 AI 帮你解释概念、梳理公式和定位依据。

## 下载体验

Windows 打包版可在 Release 页面下载：

- [PDF AI Reader v1.0.0](https://github.com/Richardwongyk/PDF-AI-Reader/releases/tag/v1.0.0)
- [直接下载 Windows 压缩包](https://github.com/Richardwongyk/PDF-AI-Reader/releases/download/v1.0.0/PDF-AI-Reader-Windows.zip)

下载后请先完整解压 `PDF-AI-Reader-Windows.zip`，再运行解压目录中的 `PDF-AI-Reader.exe`。
不要在压缩包预览窗口里直接双击 exe，也不要把 exe 单独移出文件夹；程序运行需要旁边的 `app` 和 `runtime` 目录。

## 它能做什么

### 流畅读论文

打开 PDF 后直接进入阅读状态。页面、目录、缩放、滚动和段落交互都围绕长篇论文阅读设计，重点是让读者尽快回到正文，而不是先处理工具本身。
界面提供深色、米黄、浅灰、黑白四套主题，并在侧栏、输入框、表格和右侧回答区保持足够对比度。翻译框与批注框采用独立配色，方便在长时间阅读中区分不同类型的内容。

应用会在本地保存阅读会话，包括上次打开的标签、标签顺序、当前激活文档、页号和页内滚动位置。下次启动时会自动恢复阅读现场；如果对应文件被删除或内容被替换，则安全跳过位置恢复，不影响正常打开。

主阅读区顶部提供类似浏览器的阅读标签条。打开新的 PDF 时，如果当前标签已经有文档，会自动创建新标签；也可以用 `Ctrl+T` 或标签条右侧 `+` 新建空白标签。每个标签保存自己的文档入口和阅读位置，拖拽排序后也会按界面顺序保存，切换时自动恢复目标标签的位置。

应用采用单实例启动：重复运行 `run_py314.bat` 或静默脚本不会再开第二个窗口，只会激活已有主窗口；如果通过命令行携带 `--open` 文件路径，已有主窗口会用新标签打开该 PDF。

### 双击就翻译

遇到难读的段落，双击即可翻译。翻译会尽量保留论文里的公式、符号和上下文，让你不必在阅读器、翻译网页和笔记软件之间来回切换。

### 对整篇论文提问

你可以问：

- 这篇论文解决了什么问题？
- Self-Attention 为什么能替代循环结构？
- 这个公式里的每个符号是什么意思？
- 作者在哪一页解释了实验设置？

回答会尽量基于当前 PDF 的内容，而不是泛泛聊天。

### 带着证据回答

论文阅读最怕"看起来很对"。PDF AI Reader 的目标是让回答能回到原文：页码、段落、公式和上下文都应该可以追溯，方便你确认 AI 没有偏离论文。

### 更认真地处理公式

很多论文的关键都藏在公式里。PDF AI Reader 会在后台索引公式内容，结合 PDF 结构信息、本机模型和证据校验，让数学内容不只是被当成普通文本处理。

### 为深度阅读而生

它适合阅读机器学习、数学、物理、计算机系统等公式密集、上下文复杂的论文。无论是快速扫读一篇 paper，还是反复研究一篇经典论文，都可以把阅读、翻译、问答和证据放在同一个界面里。

## 已实现功能

核心功能包括：
- PDF 阅读（渲染、缩放、目录导航、页码跳转）
- 段落翻译（双击即译，流式输出，缓存加速）
- 全文问答（知识库检索 + 证据追溯）
- 概念解释（右键段落一键解释）
- 公式识别（后台自动索引 + 本机结构模型）
- 四主题切换（深色/米黄/浅灰/黑白，菜单栏一键切换）
- 单窗口阅读标签（顶部标签条新建、切换、拖拽排序和关闭多个阅读入口）
- 恢复上次阅读会话（关闭后重新打开会按原数量和顺序恢复标签，并回到上次激活标签的位置）
- 静默启动（`.vbs` 双击无命令行窗口）

## 适合谁

- 经常读英文论文的学生和研究者
- 想快速理解经典论文的开发者
- 需要反复查证公式、定义和实验细节的读者
- 希望把 AI 用在严肃阅读，而不是只做摘要的人

## 使用指南

### 安装与启动

1. **安装依赖**（需要 Python 3.14.4）：

   ```powershell
   # 创建 conda 环境并安装依赖
   conda create -n pdf_ai_reader_314 python=3.14.4
   conda activate pdf_ai_reader_314
   pip install -r requirements.txt
   ```

2. **配置模型**：复制 `config.example.yaml` 为 `config.yaml`，填写你的 API Key 和模型偏好：

   ```yaml
   model:
     local: "qwen3.5:4b"          # 本地 Ollama 模型（翻译/问答）
     cloud: "deepseek/deepseek-v4-flash"  # 云端模型（翻译/问答）
     embedding: "bge-m3"           # 嵌入模型（知识库检索）
     ollama_host: "http://localhost:11434"
   api_keys:
     deepseek-v4-flash: "sk-your-key-here"
   ```

3. **启动应用**：

   ```powershell
   python src\main.py
   ```
   或双击 `run_py314.vbs` 静默启动（无命令行窗口），`run_py314.bat` 可见日志。

### 基本操作

| 操作 | 方式 |
|------|------|
| 打开 PDF | 菜单栏 **文件 → 打开 PDF**，或点击工具栏 📂 按钮 |
| 翻页/滚动 | 鼠标滚轮 或 触控板双指滑动 |
| 缩放 | `Ctrl+=` 放大，`Ctrl+-` 缩小 |
| 页码跳转 | 工具栏输入页码后按 Enter |
| 目录跳转 | 左侧导航栏点击章节标题 |
| 隐藏工具栏 | 双击工具栏空白区域 |
| 显示工具栏 | 双击顶部蓝色细条 |

### 翻译段落

**双击 PDF 中的任意段落**，该段落下方会"裂开"显示翻译结果。

- 翻译会自动保留公式、符号和上下文。
- 再次双击可折叠/展开翻译。
- 点击 **⟳ 重新翻译** 可绕过缓存重新请求。

### 全文问答

在右侧 **AI 工具集**面板中输入问题，AI 会检索全文知识库后回答：

- 问题示例：
  - "这篇论文解决了什么问题？"
  - "作者在哪一页解释了实验设置？"
  - "这个公式里的每个符号是什么意思？"

- **证据面板**：检索到的原文片段会列出页码和相关度评分。
- **双击证据项**可跳转到原文对应位置。
- **追问按钮**：回答完成后自动生成建议问题，一键追问。

### 概念解释

**右键段落 → 请解释**，AI 会结合上下文解释该段落中的核心概念。

### 术语表管理

菜单栏 **工具 → 术语表管理器** 可维护翻译术语。编辑器支持按领域查看、搜索、新增、删除、导入 JSON/CSV、设置强制翻译、填写别名和备注；保存后会立即刷新当前翻译服务。

### 公式处理

- **自动索引**：打开 PDF 后，应用会在后台索引公式内容，不阻塞阅读。
- **本机模型优先**：带文本层的数字 PDF 会优先使用字符、坐标、字体和本机结构模型证据；图片或扫描公式则走更谨慎的识别路径。
- **结果保护**：不确定的公式结果不会直接覆盖正文或写入知识库，只有可信结果才会进入后续问答和检索流程。

### 主题切换

菜单栏 **视图 → 主题** 即可切换，自动保存到 `config.yaml`：

| 选项 | 风格 | 强调色 |
|------|------|--------|
| `dark` 深色 | 黑底 + 高对比文字 | 蓝色 |
| `sepia` 米黄 | 暖米黄底 + 深灰蓝字 | 琥珀 |
| `light_gray` 浅灰 | 浅灰底 + 深灰字 | 浅蓝 |
| `black_white` 黑白 | 高对比黑白阅读风格 | 靛紫 |

也可直接在 `config.yaml` 中修改 `ui.theme`。

### 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl+O` | 打开 PDF |
| `Ctrl+=` / `Ctrl+-` | 放大 / 缩小 |
| `Ctrl+Enter` | 发送提问（在输入框中） |
| `Esc` | 折叠当前展开的翻译/问答 |
| 双击段落 | 翻译 / 折叠切换 |
| 双击工具栏空白 | 隐藏工具栏 |
