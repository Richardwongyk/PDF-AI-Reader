# PDF AI 阅读器 · 极简设计报告

## 产品目标

理工科学生/研究者读英文论文用的桌面工具。功能：

1. **PDF 阅读**：打开 PDF，滚动翻页，数学公式正确识别保留
2. **段落翻译**：双击段落在其下方显示 AI 翻译，术语按内置表强制译法
3. **段落问答**：选中段落后提问，AI 结合全文向量库回答
4. **目录与书签**：侧边栏章节大纲，手动/AI 书签

翻译和问答优先使用本地模型（Qwen3.5:4b，Ollama 运行），也可配云端模型（DeepSeek）。

## 三个核心流程

**打开 PDF**：PyMuPDF 读出文本块 → 按坐标合并为自然段 → 两轮公式检测（字体/LaTeX 特征启发式 → Pix2Text ONNX 精扫）→ 传给 PdfViewer 按页占位，只渲染可见 ±1 页 → 后台 BGE-M3 分段嵌入写入 ChromaDB（每批 50 条），相同文件（SHA256）下次直接加载。

**翻译段落**：双击段落 → 下方插入 SplitWidget 裂缝容器 → 将 `$...$` 替换为 `【FORMULA_N】` 占位符 → 组装 Prompt（角色 + 公式保护指令 + 术语表 + 定理约束 + 2 组对照样例）→ 流式调 LLM → 还原占位符 → QWebEngineView 渲染 Markdown + KaTeX。

**段落问答**：输入问题 → 向量库检索 top-3 相关段落 → 组装上下文（当前段 + 检索段 + 最近 6 轮对话）→ 流式调 LLM → 回答显示在裂缝中，结束后生成 3 个追问建议。

## 关键技术选型

| 组件 | 选型 | 原因 |
|------|------|------|
| GUI | PySide6 | Qt 绑定，信号/槽 |
| PDF 引擎 | PyMuPDF | 速度快，直接拿文本坐标 |
| 本地 LLM | Qwen3.5:4b (Ollama) | 4B 量化后 ~2.7GB，16GB 内存够 |
| 嵌入模型 | BGE-M3 (1024维) | 多语言向量 |
| 向量库 | ChromaDB | 嵌入式，无需部署 |
| 云端网关 | LiteLLM | 统一接口，内置重试 |
| 公式渲染 | KaTeX | 比 MathJax 轻 |

## 异常处理

无 API Key 用 MockLLMClient 降级，不崩溃。LLM 调用失败 → 裂缝显示错误提示。PDF 损坏/加密 → 弹对话框，不崩应用。全局 `sys.excepthook` 兜底。

## 线程

| 线程 | 职责 |
|------|------|
| 主线程 | Qt 事件循环、UI 绘制 |
| _ParseThread | PDF 打开、分块、公式精扫 |
| QThreadPool（≤2） | 知识库构建 |
| _TranslationThread | 每次翻译一个 |
| _QAThread | 每次问答一个 |
